# Editor Engine Specification

## 1. Core Requirements

| ID | Requirement Description | Keyword |
| :--- | :--- | :--- |
| ED-1 | Centered, max-width layout (600px - 800px) with custom fonts (Serif/Sans-serif), size, and spacing. | MUST |
| ED-2 | Editor becomes read-only and ignores user keyboard/mouse typing inputs during streaming generation. | MUST |
| ED-3 | Newly generated AI text MUST be styled with a distinct CSS highlight (golden/violet). | MUST |
| ED-4 | Highlight is removed on user keystroke, focus change, or explicit click on "Accept". | SHALL |
| ED-5 | Regeneration discards the last AI generation slice and triggers a new completion. | MUST |
| ED-6 | Undo action restores the editor to the state immediately preceding the last generation. | MUST |

## 2. Scenarios

### Scenario 1: Streaming Generation Lock
- **Given** the user is typing in the editor
- **When** the "Generate" command triggers
- **Then** the editor MUST enter read-only mode and ignore all keyboard inputs until generation terminates

### Scenario 2: Highlight Removal on Activity
- **Given** streaming generation completes with a highlighted text block
- **When** the user presses any key or clicks outside the editor
- **Then** the highlight MUST immediately disappear

### Scenario 3: Regenerate Action
- **Given** a story with the last block generated by AI
- **When** the user clicks "Regenerate"
- **Then** the last AI block MUST be removed and a new streaming request sent to the API
